main() 메서드를 가진 파일에서 클래스명은 파일명과 동일해야 한다.
클래스명은 대문자로 시작한다. =파일명과 클래스명이 다르면 자바 프로그램이 실행되지 않는다.
class Exam01 {
  public static void main(String[] args) {
   
  }
}
자바 클래스의 기본형이다. java 코드는 모두 이 기본형을 만들고 작성한다.
ls는 리눅스에서 파일 목록을 읽어주는 명령어이다.
자바 소스 파일을 자바 개발 도구(JDK) javac 컴파일러로 컴파일(번역)-> ~.class 바이트 코드 생성->자바로 바이트 코드 실행-> 자바 프로그램 실행
소문자로 시작하면 변수명(카멜 표기법을 사용하지만 첫 글자를 소문자로 적는다. 클래스의 이름은 첫 글자를 대문자로 적는다는 규칙과 겹치지 않게 하기 위함이다.)
대문자로 시작하면 클래스명(ex. Exam01)
변수명만 보고 그 안에 담긴 값의 종류를 생각할 수 있도록 작명하지만 실수로 다른 형태의 값을 담거나 할 수도 있으므로, 자바에서는 변수의 특징을 이름 앞에 표시해서 그 형태를 지정해준다. 따라서 한번 정해준 형태와 다른 형태의 값을 변수에 담을 수 없다.(ex. int, double, String)
변수의 성격을 정해주므로 성격과 불일치하는 값을 넣을 시 컴파일할 때 에러가 뜨게 된다.
이때 통합 개발 도구 이클립스를 쓰면 컴파일하지 않더라도 편집기창에서 다른 형태의 값이 변수에 대입되었다고 에러 표시를 보여준다.
